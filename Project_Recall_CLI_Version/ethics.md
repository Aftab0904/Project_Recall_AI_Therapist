# Ethical Guidelines: Project Recall

## Privacy & Consent
- User data must be stored locally whenever possible.
- Sensitive identifiers should be minimized.
- Users must be informed that memories are being extracted to provide better care.

## Safety & Non-Maleficence
- Memories retrieved for session openings must be filtered for potential triggers.
- High-risk topics (e.g., self-harm, severe trauma) should be handled with extra care and not used in automated re-engagement without human oversight.

## Transparency
- The AI should be transparent about using past information to maintain continuity.
- "I remember you mentioned..." should be used to build rapport, not to mimic human consciousness.

## Accountability
- The system provides a scoring mechanism for safety and importance, allowing for auditing of automated retrievals.
